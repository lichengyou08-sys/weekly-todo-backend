# 每周待办事项 - 云端版

一个支持跨设备同步的每周待办事项管理应用，帮助您高效管理每周任务。

![版本](https://img.shields.io/badge/版本-1.0.0-blue)
![技术栈](https://img.shields.io/badge/技术栈-Node.js%20%2B%20Express%20%2B%20SQLite-green)

## 功能介绍

### 核心功能

| 功能 | 说明 |
|------|------|
| **用户系统** | 注册/登录账号，数据与账号绑定 |
| **每周视图** | 周一到周日七天任务管理 |
| **任务优先级** | 高/中/低三级优先级，用不同颜色区分 |
| **重复任务** | 支持设置每周重复的任务 |
| **完成追踪** | 勾选完成任务，自动记录完成历史 |
| **本周总览** | 左侧边栏显示本周所有任务，支持折叠 |
| **数据统计** | 实时显示总任务数、已完成、待完成数量 |
| **进度条** | 可视化展示本周任务完成进度 |
| **深色模式** | 支持浅色/深色主题切换 |
| **响应式设计** | 完美适配电脑、平板、手机 |

### 任务卡片说明


每个任务显示：
- **复选框** - 点击标记完成/未完成
- **任务内容** - 任务描述文字
- **🔄 图标** - 表示这是每周重复的任务
- **优先级标签** - 高(粉色)/中(橙色)/低(蓝色)
- **删除按钮** - 删除该任务

### 添加任务

1. 在输入框输入任务内容
2. 选择优先级（高/中/低）
3. 选择是否每周重复
4. 点击 **+** 按钮或按 **Enter** 添加

### 本周总览

- **可折叠** - 点击标题可以展开/收起
- **显示所有任务** - 本周七天的任务集中展示
- **实时同步** - 添加/完成任务后自动更新

### 完成记录

- 点击右下角 **📊** 按钮查看
- 可按本周/本月/全部筛选
- 显示任务内容和完成时间

## 技术栈

### 后端
- **Node.js** - JavaScript 运行时
- **Express** - Web 框架
- **SQLite3** - 轻量级数据库
- **bcryptjs** - 密码加密
- **express-session** - 用户会话管理

### 前端
- **HTML5** - 页面结构
- **CSS3** - 样式设计（马卡龙配色）
- **JavaScript** - 交互逻辑
- **Fetch API** - 后端通信

### 部署
- **Cloudflare Tunnel** - 内网穿透（开发测试）
- 支持部署到 Railway、Render、腾讯云等平台

## 本地运行

### 环境要求
- Node.js 16.0.0 或更高版本

### 安装步骤

1. **克隆仓库**
   ```bash
   git clone https://github.com/你的用户名/weekly-todo-backend.git
   cd weekly-todo-backend
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动服务器**
   ```bash
   npm start
   ```

4. **访问应用**
   浏览器打开 http://localhost:3000

### 快速启动（Windows）

双击 `start.bat` 文件，自动启动服务器和 Cloudflare Tunnel。

## 数据说明

### 数据存储
- 用户数据、任务数据存储在 `database.db`（SQLite 数据库）
- 数据库文件在本地，重启电脑数据不丢失
- 建议定期备份 `database.db` 文件

### 跨设备同步
- 使用同一账号登录，数据自动同步
- 数据存储在服务器端，换设备登录即可看到相同数据

## 部署方案

### 方案 1：本地 + Cloudflare Tunnel（当前方案）
- **优点**：免费、简单、数据在自己电脑
- **缺点**：电脑关机后无法访问，每次重启链接会变
- **适合**：个人使用，电脑长期开机

### 方案 2：Railway（推荐）
- **优点**：24小时在线、自动 HTTPS、稳定
- **缺点**：需要绑定信用卡验证（不扣费）
- **适合**：需要随时随地访问

### 方案 3：腾讯云/阿里云
- **优点**：国内访问快、有免费额度
- **缺点**：需要实名认证
- **适合**：国内用户长期使用

## API 接口

| 接口 | 方法 | 说明 | 认证 |
|------|------|------|------|
| `/api/register` | POST | 用户注册 | 否 |
| `/api/login` | POST | 用户登录 | 否 |
| `/api/logout` | POST | 退出登录 | 是 |
| `/api/check-auth` | GET | 检查登录状态 | 否 |
| `/api/tasks` | GET | 获取任务列表 | 是 |
| `/api/tasks` | POST | 创建新任务 | 是 |
| `/api/tasks/:id` | PUT | 更新任务 | 是 |
| `/api/tasks/:id` | DELETE | 删除任务 | 是 |
| `/api/history` | GET | 获取完成历史 | 是 |
| `/api/stats` | GET | 获取统计数据 | 是 |

## 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `PORT` | 服务器端口 | 3000 |
| `SESSION_SECRET` | Session 加密密钥 | your-secret-key-change-this |
| `DB_PATH` | 数据库文件路径 | ./database.db |
